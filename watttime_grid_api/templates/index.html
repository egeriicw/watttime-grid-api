Why WattTime Impact API?

The CO2 intensity of electricity varies in real time throughout the day. Ever wanted to know...

  • What time of day has the cleanest energy in New England?
  • What is the carbon footprint of my electricity, right here, right now?
  • How variable is wind power production in the Midwest?
  • Is California on track to meet its 33%-by-2020 renewable portfolio standard?

It turns out that for most parts of the US, the data to answer these kind of questions are freely and openly available. But these data are scattered in wildly different formats across a wide variety of sources, not all of which are exactly easy to find.

The WattTime Impact API standardizes this information to empower app developers to build cool tools that make real-time electricity data accessible, exciting, and actionable. It's also useful for data analysts who want a single portal that provides energy cleanliness data in a simple, queryable, standardized format.

What data are available?

Geographically, the data are organized the same way the grid is: into "balancing authorities" that manage supply and demand for a given service area. The bigger balancing authorities, called Independent Services Operators and Regional Transmission Organizations (ISOs/RTOs), together cover about 2/3 of US electricity consumers.

The API provides data for nearly every ISO/RTO region in the country (and metadata for many others):

Tip: You can find out what ISO/RTO a user is in by providing geospatial parameters to the balancing_authorities endpoint.

As befits a real-time data service, there are also time-series oriented dataâ€”what's happening on the regional grid at a particular moment? In addition to metadata about data provenance, two kinds of data types are provided for each time series data point:

Generation mix
The power on the grid is generated from different fuel sources at different times; sometimes there's more coal, other times there's more wind. (We count wind and other renewables as types of fuels for ease of comparison.) The API provides the number of megawatts generated from each fuel. Pie charts or streamgraphs, anyone?
Carbon footprint
Because the generation mix is constantly changing, the carbon impact of electricity use is constantly changing too. The API provides carbon emissions intensity in pounds of CO2 per megawatt-hour for each fuel typ. Combine with a user's Green Button data for a high-resolution localized carbon footprint analysis!
Tip: You can select specific date ranges by providing start and end time parameters to the datapoints endpoint.

Where do the data come from?

Power generation data is pulled straight from the ISOs in real time, as frequently as they release it: every 5 minutes in ISONE, MISO, PJM, and BPA; every 10 minutes in CAISO; and every hour in ERCOT. Plus, every morning, SPP releases yesterday's 5-minutely data, and CAISO releases yesterday's detailed hourly data. We combine all this with CO2 emissions factors from the Energy Information Administration to calculate the carbon impact of energy use in each place at each time.

In a few cases we have to use approximations if a region combines two fuel types. For example, BPA's data focuses primarily on its substantial renewable energy resources and doesn't distinguish between different types of thermal power plants. These approximations are all clearly marked in our data stream. Stay tuned for further improvements in our ability to disaggregate different fuel types.

How can I use the API?

Get started by checking out the interactive documentation and the user group. It's a standard RESTful JSON API, so you can make HTTP requests to API endpoints using any of your favorite tools, from iOS to d3.js to curl.

Right now, the API is completely free and open: no authorization, no rate limits, no fees. Please be respectful of our limited server resources. Data availability terms are subject to change without notice, although of course we'll do our best to keep the user group informed about any upcoming changes.
